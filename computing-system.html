1.1

Evolution of Computing

Computing devices have transformed from basic calculators to advanced data processors in just a few decades. The Abacus is considered the first known calculating tool — beads on rods performing addition and subtraction, used across cultures for thousands of years.

The pioneers

Note the year + one key fact for each pioneer
1837

Charles Babbage

Designed a mechanical general-purpose computer — the Analytical Engine. Never fully built in his lifetime, but the design laid the foundation for modern computing. Known as the Father of the Computer.

1843

Augusta Ada Lovelace

Wrote the first algorithm intended for a machine (Babbage's Analytical Engine) — earning her the title of the first computer programmer.

1890

Herman Hollerith

Invented a punch-card-based machine used in the U.S. census, marking the shift toward mechanised data processing.

1945

John Von Neumann

Published the stored-program architecture — instructions and data sharing one memory unit. Still the blueprint for almost every modern computer.

The Microprocessor

A microprocessor — CPU logic on a single IC

NOTE
A microprocessor is a computer processor where the data processing logic and control are included on a single integrated circuit (IC), or a microchip.

CPU vs. Microprocessor — what's the difference?

Note the two definitions below

CPU

The job title. The part of a computer that processes instructions — the "brain." Every computer needs one.

Microprocessor

The tiny chip that does that job. A CPU built entirely onto a single integrated circuit (IC).

SIMPLE ANALOGY

Every computer has always had a CPU. In the earliest computers (1940s–60s), that "brain" was huge — built from thousands of separate vacuum tubes and wires, filling an entire room. It was still a CPU, just a massive one. Then in 1971, Intel packed that entire brain onto one tiny chip — the Intel 4004. That single chip is a microprocessor.

NOTE — exam line
Exam-ready line: A microprocessor is a CPU built on a single integrated circuit. Older CPUs were built from many separate components — the microprocessor made it possible to fit the whole CPU onto one small chip.

Moore's Law & chip density

Gordon Moore (1965, revised 1975): the number of transistors that can be economically placed on an IC roughly doubles every two years.

1.2.2

CPU & Registers

The CPU is the brain of the computer — it fetches, decodes, and executes instructions. Explore its parts below.

NOTE

The Control Unit (CU)

Acts like a "traffic policeman" inside the computer. It directs the flow of data between the different functional units and interprets the instructions written in a program — but performs no calculations itself.

NOTE

Arithmetic Logic Unit (ALU)

Performs every arithmetic operation (addition, subtraction, multiplication, division) and every logical operation (greater than, equal to, less than). This is the component that plays the main role in decision‑making inside the CPU.

NOTE

Registers — the CPU's fastest memory

Registers are the smallest and fastest storage locations inside the CPU, temporarily holding data, instructions, and addresses during execution.

1.3

Software

Hardware is nothing without instructions. Software is broadly split into System Software (manages hardware — OS, utilities, drivers) and Application Software (does a job for the user). A computer cannot run without system software — but it can still boot and idle without any application software installed.

System software is the set of programs that controls and manages the hardware, acting as the bridge between the machine and the user's applications. It's split into three families:

The interface between user and hardware — the main control program that manages every other program. It's the first software loaded into memory when a system boots.

Application Software

Note the two types + two examples each

Often just called an "app" — a program designed to perform a specific task for the user. It sits between the user and the system software.

General‑purpose

Built for generic, everyday tasks used across almost any field. Examples: Word Processors, Spreadsheets, Presentation software, DBMS, Web Browsers, Graphics tools.

Special‑purpose

Built exclusively to perform one specific job. Examples: Accounting Software, Reservation Systems, Inventory Control, Payroll Systems, Media Players.

NOTE — exam line
Exam-ready line: A computer system cannot function without system software, but it can still run without application software — without an OS a device won't boot at all, but without a calculator app it will simply do less.

Programming Languages & Translators — tap a level

Note whether each needs a translator, and which one

Computers only understand machine language — binary 1s and 0s. Programmers write in something more human‑friendly, so a translator converts it back into machine code.

NOTE

Machine Language — Low‑level, machine‑dependent

Instructions written directly as 1s and 0s. Directly understood and executed by the computer, so no translator is needed — but writing and debugging it (remembering opcodes, character codes, addresses) is extremely difficult for humans.

NOTE

Assembly Language — Low‑level, machine‑dependent

Uses English‑like symbolic notations (mnemonics) instead of 1s and 0s, making code easier to write. Still totally machine‑specific — code for one machine won't run on another. Needs a translator called an Assembler to convert it into machine language.

TWO KEY TERMS

Opcode (Operation Code): the portion of a machine‑language instruction that specifies the operation to perform — the only form a CPU's control unit directly understands, always in binary.

Mnemonic: a short, memorable name standing in for an opcode in assembly language — e.g. ADD, MOV, JMP — instead of memorising strings of binary digits.

NOTE

High‑Level Language — Machine‑independent

Uses English‑like sentences, and simplifies coding a great deal. Not directly understood by the computer, so it needs a translator — either a Compiler or an Interpreter.

Compiler

Translates the entire program into machine code all at once, before execution. Examples: GNU Compiler Collection (GCC), Java compiler.

Interpreter

Translates and executes the program line by line, during runtime. Examples: Python interpreter, BASIC interpreter.

Program Development Tools

Writing code needs a text editor to create and save the source code file — lightweight coding editors add syntax highlighting and code completion. To simplify things further, an Integrated Development Environment (IDE) bundles the editor, compiler/interpreter, and debugger into a single screen. Examples: Python IDLE, Geany, NetBeans, Eclipse, Atom, Lazarus.

Free & Open‑Source Software (FOSS)

FOSS grants users the freedom to run, study, change, and share software and its source code. We use it every day: Operating Systems (Ubuntu, Fedora, Android — all Linux‑kernel based), Browsers (Firefox), Office Suites (LibreOffice), Graphics (GIMP, Blender), Web Servers (Apache HTTP Server), and Media Players (VLC).

Open source vs. closed source

Note all four comparison rows, both columns

Open‑Source

  • Source code: open — you can view, study, and modify it
  • Updates: a global community collaborates; fixes can be rapid
  • Security: "many eyes" find and fix vulnerabilities quickly
  • Customization: high — change it to suit your needs
  • Examples: Ubuntu, Firefox, LibreOffice, GIMP, Blender, Apache

Closed‑Source (Proprietary)

  • Source code: kept secret by the owning company
  • Updates: only the developing company can apply fixes
  • Security: obscure — you must trust the vendor
  • Customization: low — wait for the company to add a feature
  • Examples: Windows, macOS, iOS, MS Office, Photoshop
NOTE — exam line
Free ≠ Open Source (subtle distinction): they usually overlap, but the free software movement is driven by social justice and user liberty, while the open source movement is driven by practical, better software development.

The Father of Free Software

Note his 3 landmark contributions
b. 1953

Richard Stallman (RMS)

American programmer who began at the MIT AI Lab's "hacker culture" of freely‑shared code. When software started being privatised in the early 1980s, he found it ethically wrong — leading to three landmark contributions: The GNU Project (1983) — a complete, free Unix‑like OS ("GNU is Not Unix"); the Free Software Foundation (1985); and the GNU General Public License (GPL), which introduced Copyleft — using copyright law itself to keep software free to use, modify, and redistribute.

1.5

Green Computing & E‑Waste

Green computing designs, uses, and disposes of computing devices in an eco‑friendly, energy‑efficient way. E‑waste contains both valuable metals (gold, silver, copper) and hazardous substances (lead, mercury, cadmium) — improper disposal poisons soil and groundwater.

Disposal methods — safe or unsafe?

Note 3 safe + 4 unsafe methods, one line each

Reuse

If a device still works, sell it or donate it to someone who needs it.

Recycle

Send non‑working devices to a certified e‑waste recycler to safely recover metals and plastics.

Manufacturer take‑back

Companies like Apple, Dell and HP offer free return/trade‑in programmes.

Land filling

E‑waste buried in landfills lets toxic chemicals leach into the surrounding environment.

Incineration

Burning e‑waste reduces its volume but releases harmful toxins and gases into the air.

Acid baths

Informal recyclers dissolve metals in strong acids — dangerous, and pollutes local water sources.

Exporting to developing countries

Illegal export to countries with poor regulation harms workers and the environment.
