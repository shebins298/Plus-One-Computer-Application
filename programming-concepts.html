Programming Concepts

A computer has no common sense — it only follows precise, step-by-step instructions. This chapter walks through the full journey from a vague idea to a working, tested, documented program.

"The function of a good software is to make the complex appear to be simple." — Edward Bernard

3.1

Introduction

Software is a collection of computer programs — sets of instructions written in a language a computer can understand and execute to solve specific problems. Because computers lack the ability to make assumptions, they require precise, step-by-step instructions to carry out even the simplest operations.

Writing a computer program is not merely about typing commands. It requires a clear understanding of the problem, the design of a logical solution, and the translation of that solution into a programming language. This holistic process is known as Program Development.

3.2 Problem Solving Using Computers

We solve problems every day — a math puzzle, a travel route, even making tea — by naturally breaking the task into smaller steps. Computers approach problems the same way, but with one major difference: humans have common sense, computers do not. A computer is incredibly fast and never tires, but it cannot think or guess. We act as the planner, figuring out the solution in our mind, and then direct the computer to execute that plan.

3.3

Writing a program is like constructing a building — you cannot start stacking bricks without a plan first. Programming involves two kinds of thinking: Analytical Thinking (breaking a big problem into smaller, easier parts) and Creative Thinking (finding smart, efficient ways to solve those parts).

3.3.1

Problem Identification

Problem identification is the foundation of the entire programming process — clearly defining the gap between the current state (the problem) and the desired state (the solution). To identify a problem effectively, we look for three key components:

  1. What raw data or facts do we have?
  2. What is the final result we want the computer to produce?
  3. How do we transform the data into the result?

Applied to our example

Vague idea: "Calculate total marks for a student." — too vague for a computer.

1. Data Needed: marks (out of 100) for six subjects — M1, M2, M3, M4, M5, M6.

2. Desired Result: the total marks and the percentage of marks.

3. The Process:

Total = M1 + M2 + M3 + M4 + M5 + M6
Percentage = (Total / 600) × 100

Final Problem Statement

Create a program that accepts marks of six subjects as input, calculates their sum and percentage using standard formulas, and displays both results.

3.3.2a

Algorithms

An algorithm is a step-by-step procedure to solve a problem — like a recipe for cooking, or directions to a friend's house. The word comes from the Persian mathematician Muhammad Ibn Musa Al-Khwarizmi, who lived around 820 AD and is often called the father of algebra.

Guidelines for writing algorithms

1. Storing Information: Computers need named storage spaces (like Marks, Age, Result) to hold data while working on it.

2. Writing the Steps:

• Input: use Input, Read, or Get — e.g. Input Number1

• Process: use Calculate, Add, or Set — e.g. Calculate Total = Number1 + Number2

• Output: use Print, Display, or Show — e.g. Print Total

3. General Rules: every algorithm must Start and Stop; be Clear (unambiguous); use Simple Steps (one action per step); and Must End — it cannot run forever.

Real-life example — making lime juice

Step 1: Start Step 2: Get Lime, Sugar, Water, Ice — Input Step 3: Cut Lime — Process Step 4: Squeeze Lime into Glass — Process Step 5: Add Sugar and Water — Process Step 6: Stir Well — Process Step 7: Serve Juice — Output Step 8: Stop

SEQUENTIAL CONTROL FLOW

Problem: Calculate area and perimeter of a rectangle, given length and breadth.

Step 1: Start Step 2: Input Length, Breadth Step 3: Perimeter = 2 x (Length + Breadth) Step 4: Area = Length x Breadth Step 5: Print Perimeter, Area Step 6: Stop
BRANCHING / SELECTION

Problem: Compare two test scores (M1, M2) and display the higher one.

Step 1: Start Step 2: Input M1, M2 Step 3: If M1 > M2, then Go to Step 4 else Goto Step 5 Step 4: Print "M1 is higher". Goto Step 6 Step 5: Print "M2 is higher". Step 6: Stop
ITERATION / LOOPING

Problem: Print the first 10 multiples of a number N.

Step 1: Start Step 2: Read N Step 3: Set A to 1 Step 4: Calculate M as A × N Step 5: Display M Step 6: Increase A by 1 Step 7: Repeat Steps 4 to 6 while A is less than or equal to 10 Step 8: Stop
COMBINED — OUR RUNNING EXAMPLE

Problem: Take marks of six subjects, find the total, and calculate the percentage.

Step 1: Start. Step 2: Input M1, M2, M3, M4, M5, M6. Step 3: Total = M1 + M2 + M3 + M4 + M5 + M6. Step 4: Max = 600. Step 5: Percentage = (Total / Max) x100. Step 6: Display Total, Percentage. Step 7: Stop.

Flow of control — three ways a program can move

1. Sequential (the straight line): every instruction executes in the exact order written, top to bottom — like reading a book, or baking a cake in order.

2. Branching / Selection (the crossroad): the program checks a condition and follows one of two paths — like a traffic light, represented with 'If... Then... Else' logic.

3. Iteration / Looping (the cycle): a set of instructions repeats as long as a condition remains true — like climbing a flight of stairs one step at a time.

3.3.2b

Flowcharts

A flowchart is a pictorial representation of an algorithm. Just as a map helps you navigate better than written directions, a flowchart helps programmers visualize the flow of control using standard geometric shapes connected by arrows.

Standard flowchart symbols

Terminator

Marks the beginning (Start) or end (Stop) of a process.

Process

Represents a process, task, or action to be performed.

Decision

Indicates a decision or branching point (Yes/No, True/False).

Input / Output

Represents input to or output from a process.

Flow line

An arrow showing the direction of flow between steps.

Connector

Connects or links different sections of a flowchart.

Advantages & limitations

Advantages

Better Communication (a universal visual language) · Effective Analysis (spots dead ends and missing steps early) · Proper Synthesis (shows how modules connect) · Efficient Coding (the flowchart becomes a blueprint, making coding a simple translation task).

Limitations

Time-Consuming (drawing a neat flowchart takes longer than writing text) · Difficult to Modify (a logic change often means redrawing the whole diagram).

3.3.3

Coding

A computer's native language is Binary / Machine Language — 0s (Off) and 1s (On). Humans communicate in Natural Languages (English, Malayalam, Hindi) that are too vague and complex for a computer's strict logic. High-Level Languages (HLL) — like Python, Java, C++ — bridge the two, letting programmers write instructions using familiar words like print, input, if, and while.

NOTE The set of instructions written in a high-level language is called the Source Code. Coding is the process of translating your algorithm into source code. We use Python in this course because its syntax is simple, clean, and readable.

3.3.4

Translation

Source code is just text — the computer understands only machine language. A specialized program called a Language Translator converts our source code into Object Code (machine-readable binary). There are two main types.

Reads the entire source program at once, checks for errors, and translates it into a complete object code file. Once generated, the program can run multiple times without re-translating. Languages like C and C++ use compilers.

Reads the source code one line at a time, translates it, and executes it immediately before moving on. It does not generate a separate object code file — if it hits an error, it stops instantly at that line. Python and JavaScript use interpreters.

CompilerInterpreter
Translates the whole program at onceTranslates and executes the program line by line
Generates a separate object code fileDoes not generate an object code file
Scans the entire code and lists all errors at the endStops execution immediately when the first error is found

Applied: our Student Result Management program is written in Python, so the Python interpreter processes each instruction and converts it into machine-readable object code — allowing the addition and division operations to run and the final results to display.

3.3.6

Execution & Testing

Execution is simply running a program to process data. Testing is the intentional process of checking that program for errors. In short: execution proves the code works, testing proves the code is correct.

Testing our Student Result program — three kinds of test cases

Normal data

Enter marks from 50 to 100 and check the results against manual calculation.

Total = 450, Percentage = 75% ✓
Boundary data

Enter 0 or 100 for all subjects to make sure minimum and maximum values are handled correctly.

All subjects = 0, or all = 100
Erroneous data

Test for human error — what happens with invalid input?

Typing "Fifty" instead of 50 → run-time crash

This reveals that while the basic logic works, the software needs extra code to handle invalid typing. Only after passing these rigorous tests is the software considered complete and ready for documentation.

3.3.7

Documentation

Even a perfect program is useless if no one understands how to use or maintain it. Documentation is essential for three reasons: it's a memory aid for the original programmer, it enables teamwork, and it ensures easy maintenance for future developers.

Internal Documentation

Comments written directly in the source code to explain logic to other programmers. Applied: a comment explaining that total sums six specific subjects, and max_marks is set to 600 since each subject carries a maximum of 100 marks.

External Documentation

User and technical manuals providing installation and usage guides for non-programmers. Applied: a one-page User Guide covering how to open the program, that it accepts only numbers (not words) for marks, and how to read the final mark list displayed on screen.

3.4

Programming Paradigms

Just as you can travel to school by walking, cycling, bus, or car — all reaching the same destination by a different approach — programmers have developed different styles for structuring code. A programming paradigm is a fundamental style of programming that guides how the programmer thinks about and structures the code.

Treats a program like a step-by-step recipe — a large task broken into a sequence of smaller instructions or procedures. Like making tea (boil water, add leaves, pour in order), the computer executes functions one after another, top to bottom. Easy to organize in simple languages like C or Pascal, but risky for complex projects: since data is often shared across the entire program, one step might accidentally interfere with another.

Models software after real-world entities by organizing code into Objects. Each object bundles together Data (attributes — like a car's colour or fuel level) and Methods (actions — like starting the engine or accelerating). Languages like Python, Java, or C++ let developers securely hide data within objects, making OOP highly effective for complex systems like video games or operating systems — though it can be harder for beginners and needs more initial setup code.
