Data Representation

2.1

Introduction

Computers only understand binary digits (0s and 1s). Every type of data — numbers, characters, images, audio, video — must be converted into binary form before a computer can store or process it. This chapter covers number systems and conversions between them, how integers and fractional numbers are represented in binary, how characters are encoded, and how multimedia files are stored.

2.2

Number Systems

A number system is a systematic way of writing and representing numbers using a set of symbols or digits. The number of symbols used is called the base or radix of the number system.

Number systemBase (radix)Digits / symbols used
Decimal100, 1, 2, 3, 4, 5, 6, 7, 8, 9
Binary20, 1
Octal80, 1, 2, 3, 4, 5, 6, 7
Hexadecimal160–9 and A–F (A=10 … F=15)
NOTE In a positional number system, the value of a digit depends on its position, with each position representing a power of the base. The rightmost digit/bit has the smallest place value and is called the Least Significant Digit/Bit (LSD/LSB); the leftmost digit/bit has the largest place value and is called the Most Significant Digit/Bit (MSD/MSB).

Why binary?

Computers are built from transistors — tiny switches with only two states: ON (1) and OFF (0). This two-state nature is simple, reliable, and resistant to electrical noise, unlike decimal which would need 10 precise voltage levels to distinguish. Binary is used for the computer's own language (all instructions are 0s and 1s), digital electronics, efficient data storage, and encryption.

Octal & hexadecimal — where you'll meet them

2.5

Numeric Representation

Numbers without a fractional part are integers; numbers with a fractional part are floating point numbers. Computers represent each differently.

NOTE A word is a fixed-size group of bits handled as a unit by a processor. Word length is the number of bits in a word — common word lengths are 8, 16, 32, and 64 bits.

Three schemes for representing integers (8-bit examples)

Sign & Magnitude

MSB (leftmost bit) = sign → 0 = positive, 1 = negative. Remaining 7 bits = magnitude (absolute value).

−25 → magnitude 25 = 0011001, sign bit 1 → 10011001

1's Complement

Flip every bit of the true binary form (0→1 and 1→0). Used only to represent negative integers.

87 = 01010111 → 1's complement of −87 → 10101000

2's Complement

Most commonly used by computers. Steps: 8-bit binary → 1's complement → add 1.

−25 → 00011001 → 11100110 → +1 → 11100111
Exam tip — range & the "two zeros" problem: With 8 bits, Sign-Magnitude and 1's Complement can represent −127 to +127, but each has two representations of zero (00000000 and 10000000 / 11111111), which wastes a code and complicates circuit design. 2's Complement represents −128 to +127 with only one unique zero (00000000) — this is the main reason it is the standard used by computers.

Floating point representation

A floating-point number has two parts: the mantissa (the actual number, normalized) and the exponent (how many places the decimal point should move) — working like scientific notation.

NOTE 13.4 = 0.134 × 10² → mantissa = 134, exponent = 2

Character Representation

Letters, numbers and symbols also need a binary code so computers can store and display text.

ASCII

American Standard Code for Information Interchange — represents letters, numbers and symbols using 7-bit binary codes (128 characters). Published in 1963 by ASA (later ANSI). IBM's Extended ASCII uses 8 bits (256 characters).

Merits: simple and efficient; widely supported and compatible.   Demerits: limited character set; doesn't support multiple languages or special symbols well.

Unicode

Introduced because ASCII/Extended ASCII can't represent all world scripts. Maintained by the Unicode Consortium. Originally 16-bit (UCS-2, 65,536 characters); now supports up to 1,114,112 code points via UTF-8, UTF-16 or UTF-32. Benefits: universal language support, platform independence, simplified application development, efficient data sharing.

Multimedia File Formats

Images are stored as a grid of pixels, using the RGB model (Red, Green, Blue — each 0–255), typically at 24 bits per pixel. A digital image file has a Header (format, width, height, colour depth, compression), Pixel Data (actual colour values), and Metadata (date, camera settings, author, GPS, etc.).

Lossy vs. Lossless compression

Removes some data permanently to achieve a significant size reduction. The file is smaller but quality is reduced and cannot be fully restored. Used by JPEG, MP3, MP4.

Reduces file size while preserving all original data — the file can be fully reconstructed exactly as it was. Used by PNG, FLAC, BMP.
